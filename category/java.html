How to make a hello world with javafx and openjdk and Eclipse

jue 01 agosto 2019 | tags: javajavafx

The java world is changing fast. It is not easy to find a single place on how to use java12+javafx12+Eclipse (or the latest java+javafx). Here are the instructions:

Step 1: Install java

Go Here and download java for your platform (I use debian linux).

Unzip the file ( for this example I unzipped it in /home/zaq/tool/openjdk ).

In linux, you can make this java your default "java compiler for the system" with these commands:

sudo update-alternatives --install /usr/bin/java  java      /home/zaq/tool/openjdk/bin/java 100
sudo update-alternatives --install /usr/bin/javac javac     /home/zaq/tool/openjdk/bin/javac 100 
sudo update-alternatives --install /usr/bin/keytool keytool /home/zaq/tool/openjdk/bin/keytool 100

Step 2: Install Eclipse IDE

Go Here and download "Eclipse IDE for Java Developers" (currently for linux is "eclipse-java-2019-06-R-linux-gtk-x86_64.tar.gz"). Unzip the file and just execute the main program (it is just "eclipse" in linux).

Step 3: Install javaFX

Go here and download javafx 12 sdk. Nowadays javafx gets updated a lot so you will probably find javafx 13 sdk ... or 14 ... Download the SDK and unzip it somewhere. (I used /home/zaq/tool/jfx12 in my machine).

Step 4: Make a hello world program in javaFX

Create a new java project in Eclipse IDE. Get any "hello world" example from internet, there are plenty, here is one. Put the code in your java project. You will get compilation errors. Now follow these instructions to add the necessary jars for a good compilation. Also in that same page, don't forget to follow the instructions on how to make a new "launcher" configuration inside Eclipse and add the VM arguments needed to run the program.

Here are the parameteres I used for my machine:

--module-path /home/zaq/tool/jfx12/lib --add-modules javafx.controls,javafx.fxml

Now execute the launcher inside Eclipse and you should have a working "hello world" javafx12 example.

Why Eclipse and not IntelliJ?

You can of course use any IDE, but lots of people (like me) have been using Eclipse for many years and may want to keep using it.
