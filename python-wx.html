Getting started with python and wxPython

mar 01 enero 2019 | tags: pythonguidesktop

Desktop App development with Python

Instructions for creating a "hello world" desktop app with python to get you started. We cover three different GUI libraries: wxPython, GTK and QT (look for all 3 options in different pages of this blog, in this current page we cover only one of the options). Please do not use python tk because it does not produce accessible software (That is, software that can be used with a screen reader). There are already a lot of tutorials on internet about how to make desktop apps with python, but they don't tell you how to get started from zero, so we will cover that here.

General notes to help you decide which UI library to use:

  • Keep in mind that GTK is accessible in Linux, but not in Windows. Usually (if used correctly) GTK feels more natural in a Linux environment. The GNOME Desktop Environment is made with GTK.
  • wxPython is often used by blind developers.
  • QT is very mature and has extra features not available in the other libraries, like: Charts, Android deployment... Be sure to check the QT official web site for more info on what you can do with it.
  • The KDE Desktop environment uses QT.

Hello World with wxPython

How to instal wxPython in Windows

Just do a pip install like:

pip install wxpython

How to instal wxPython in Linux (tested with Debian 10 buster)

We first need to install basic development libraries

sudo apt-get install build-essential 
sudo apt-get install python3-dev
sudo apt-get install libwebkit2gtk-4.0 libjpeg-dev libtiff-dev libgtk2.0-dev libsdl1.2-dev freeglut3 freeglut3-dev libnotify-dev

Note 1 : I recommend you to create your python environment before doing the next steps

Note 2 : If you create a python environment... it is recommended you set another temporary folder for pip (if you don't you may get the error "no space left on device" while running pip install wxpython). To setup another temporary folder for pip (anywhere you have write access), just create another folder, then assign the TMPDIR environment variable to it

export TMPDIR=/another_temporary_folder

Now prepare some coffee because these next steps will take a long time to complete

pip install python-config
pip install pathlib2
pip install wxpython

To make sure wxPython is installed ok, try this code:

import wx
app = wx.App(False)  # Create a new app, don't redirect stdout/stderr to a window.
frame = wx.Frame(None, wx.ID_ANY, "Hello World!") # A Frame is a top-level window.
frame.Show(True)     # Show the frame.
app.MainLoop()

From here, the best place to go for is the wxPython github, look inside the examples and demo folders, you will get plenty of easy to understand code that you can learn from.
